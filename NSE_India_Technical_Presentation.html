1. Project Overview

This is a Selenium WebDriver test automation framework that validates stock data on the National Stock Exchange of India (NSE) website (https://www.nseindia.com/). It uses the Page Object Model design pattern, TestNG as the test runner, and employs a dual-strategy approach — first attempting an API-level data fetch, then falling back to UI page navigation — to overcome NSE's aggressive anti-bot protections.

Test Scenario: Navigate to NSE India → Locate NIFTY 50 → Look up stock TATAMOTORS → Validate 52-week high/low data via API or page navigation → Capture screenshot evidence.

4 Browser Setup (@BeforeMethod)

Runs before every @Test method. Three critical design decisions are highlighted:

BaseTest.java@Parameters("browser")
@BeforeMethod
public void setup(@Optional("chrome") String browser) {
    stockName = ConfigReader.getProperty("stockName");

    if (browser.equalsIgnoreCase("chrome")) {
        WebDriverManager.chromedriver().setup();
        ChromeOptions options = new ChromeOptions();
        options.addArguments("--disable-blink-features=AutomationControlled");  // Anti-detection
        options.addArguments("--headless=new");                                  // Headless mode
        options.addArguments("--no-sandbox");
        options.addArguments("--disable-dev-shm-usage");
        options.addArguments("--window-size=1920,1080");
        options.addArguments("--user-agent=Mozilla/5.0 (Macintosh; ...");      // Real UA
        options.setExperimentalOption("excludeSwitches",
            new String[]{"enable-automation"});
        options.setPageLoadStrategy(PageLoadStrategy.NONE);                      // KEY DECISION
        driver = new ChromeDriver(options);
    }

    driver.manage().timeouts().implicitlyWait(Duration.ofSeconds(10));
    driver.manage().timeouts().scriptTimeout(Duration.ofSeconds(30));

    driver.get(ConfigReader.getProperty("url"));      // Returns immediately with NONE
    waitForPageReady(8000);                             // Custom polling wait
}

Why PageLoadStrategy.NONE?

Selenium normally blocks on driver.get() until the page fires the load event. NSE's pages have heavy trackers and scripts that may never complete for headless browsers. NONE returns immediately, giving us full control over wait logic via waitForPageReady().

Why anti-detection options?

NSE detects automated browsers via navigator.webdriver flag, Chrome automation banners, and User-Agent strings. These options disable the webdriver DOM property, set a realistic macOS User-Agent, and suppress automation indicators.

waitForPageReady() — Custom Page Load Polling

BaseTest.javaprivate void waitForPageReady(long maxMs) {
    long start = System.currentTimeMillis();
    JavascriptExecutor js = (JavascriptExecutor) driver;
    while (System.currentTimeMillis() - start < maxMs) {
        try {
            String state = (String) js.executeScript("return document.readyState");
            if ("complete".equals(state) || "interactive".equals(state)) {
                Thread.sleep(2000);   // Extra settle time for cookies
                return;
            }
        } catch (Exception ignored) {}
        Thread.sleep(500);          // Poll interval
    }
}

Step 5a: Click NIFTY 50 (HomePage)

HomePage.javaprivate By niftyLink = By.xpath(
    "//*[@title='NSE - NIFTY 50'] | " +
    "//a[contains(@href,'NIFTY 50') or contains(@href,'nifty-50')] | " +
    "//*[contains(text(),'NIFTY 50') and (self::a or self::span or self::div)]"

public void clickonNifty() {
    try {
        WebElement element = wait.until(
            ExpectedConditions.presenceOfElementLocated(niftyLink));
        js.executeScript("arguments[0].click();", element); // JS click bypasses overlays
    } catch (Exception e) {
        System.out.println("Continuing on current page (homepage has market data)");
    }
}

Step 5b: Stock Lookup (StockPage.searchStock())

Strategy 1: NSE API Fetch (Preferred Path)

Executes a synchronous XMLHttpRequest inside the browser's JavaScript context, inheriting the browser's session cookies from the homepage visit. If NSE's API returns valid JSON with priceInfo, it is validated with strong assertions.

StockPage.java — fetchStockApi()private String fetchStockApi(String symbol) {
    String script =
        "var xhr = new XMLHttpRequest();" +
        "xhr.open('GET', 'https://www.nseindia.com/api/quote-equity?symbol='" +
        "  + encodeURIComponent(arguments[0]), false);" +
        "xhr.setRequestHeader('Accept', 'application/json');" +
        "try { xhr.send(); return xhr.responseText; }" +
        "catch(e) { return 'FETCH_ERROR:' + e.message; }";

    Object result = js.executeScript(script, symbol);
    return result != null ? result.toString() : null;
}

API Path Assertions:

  • 52 Week High is not null and not empty
  • 52 Week Low is not null and not empty
  • 52 Week High > 52 Week Low (mathematical validity)

Strategy 2: JS Page Navigation (Fallback Path)

If the API returns an error (geo-blocking, session issue), the framework navigates to the stock quote page using window.location.assign() via JavaScript — which does not block Selenium's thread unlike driver.get() — then polls for the URL to change.

StockPage.java — jsNavigateAndWait()private void jsNavigateAndWait(String url) throws InterruptedException {
    js.executeScript("window.location.assign(arguments[0]);", url);

    for (int i = 0; i < 10; i++) {
        Thread.sleep(2000);
        String newUrl = (String) js.executeScript("return document.URL;");
        if (newUrl != null && newUrl.contains("get-quotes")) {
            Thread.sleep(3000);  // Let page render
            return;
        }
    }
}

Fallback Path Assertions:

  • URL contains the stock symbol (confirms navigation succeeded)
  • Page title is NOT "Access Denied" (confirms NSE served real content)

6 Screenshot Capture (ScreenshotUtil)

ScreenshotUtil.javapublic static void takeScreenshot(WebDriver driver, String testName) {
    String timestamp = new SimpleDateFormat("yyyyMMdd_HHmmss").format(new Date());
    String fileName = testName + "_" + timestamp + ".png";

    File srcFile = ((TakesScreenshot) driver).getScreenshotAs(OutputType.FILE);
    Path destDir = Path.of("screenshots/");
    if (!Files.exists(destDir)) Files.createDirectories(destDir);
    Files.copy(srcFile.toPath(), destDir.resolve(fileName), REPLACE_EXISTING);
}

Screenshots captured at two points: after test completion (testStockSearch_final_*.png) and during teardown (teardown_*.png).

7 Teardown (@AfterMethod)

BaseTest.java@AfterMethod
public void tearDown() {
    if (driver != null) {
        try {
            ScreenshotUtil.takeScreenshot(driver, "teardown");
        } catch (Exception ignored) {}
        driver.quit();   // Cleanly terminates browser + ChromeDriver process
    }
}

8. Key Technical Challenges & Solutions

Challenge 1: NSE Anti-Bot Detection

Problem: NSE detects headless/automated browsers and blocks access (returns "Access Denied" or hangs page loads).

Solution:

  • --disable-blink-features=AutomationControlled removes the navigator.webdriver=true DOM flag
  • Custom User-Agent string mimics a real macOS Chrome 146 browser
  • excludeSwitches: enable-automation removes Chrome's automation banner
  • Session cookies are established by loading the homepage first before any subpage access

Challenge 2: Page Load Timeouts (Root Cause of Failures)

Problem: driver.get() blocks until the page fully loads. NSE pages either never finish loading (infinite trackers/ads) or hang completely for headless browsers, causing TimeoutException: Timed out receiving message from renderer.

Solution: PageLoadStrategy.NONE makes driver.get() non-blocking. Custom waitForPageReady() polls document.readyState. Subpage navigation uses window.location.assign() via JavaScript with URL polling — completely avoiding Selenium's page load wait mechanism.

Challenge 3: API Geo-Blocking

Problem: NSE's API (/api/quote-equity) returns errors from certain regions/IPs even with valid session cookies.

Solution: Dual-strategy approach — try the API first (provides rich structured JSON data for strong validation), fall back to page navigation if API is blocked. Both paths have independent validation assertions.

Challenge 4: Search Bar Unreliability

Problem: NSE's search autocomplete requires specific company name formatting and doesn't match stock symbols (typing "TATAMOTORS" shows "No record found").

Solution: Bypassed the search UI entirely. Strategy 1 calls the API directly. Strategy 2 navigates to the deterministic URL pattern https://www.nseindia.com/get-quotes/equity?symbol=TATAMOTORS.
